# 指标

# 真实指标

  • First Contentful Paint (FCP) (opens new window) —— 页面上的任何内容渲染完成的时间
  • Largest Contentful Paint(LCP) (opens new window) —— 指的是页面首次开始加载的时间开始到可视区域内最大图像或文本完成的渲染时间
  • First Input Delay(FID) (opens new window) —— 测量用户第一次与页面交互直到浏览器做出响应，并能开始处理事件经过的时间
  • Cumulative Layout Shift(CLS) (opens new window) —— 布局发生偏移

# 实验室指标

  • Total Blocking Time(TBT) (opens new window) —— 指的是FCP(首次内容绘制)与TTI(可交互时间)之前的总时间
  • Time to Interactive(TTI) (opens new window) —— TTI 指标测量页面从开始加载到完成渲染，并且能够响应用户交互的时间。

# 压缩

  • 代码压缩 UglifyJS (opens new window)
  • 文本压缩（gzip，Brotli，ZopFli）
  • Tree-shaking (opens new window)
  • Code-splitting(代码分离) (opens new window)

    常用的代码分离方法有三种：

    • 入口起点：使用 entry 配置手动地分离代码。
    • 防止重复：使用 Entry dependencies 或者 SplitChunksPlugin 去重和分离 chunk。
    • 动态导入：通过模块的内联函数调用来分离代码。

# 执行渲染

  • CSS代码优化（选择器、启用GPU、避免表达式）
  • 使用requestAnimationFrame实现视觉变化 动画
  • 降低复杂度或使用 Web Worker （service worker 基于web worker，浏览器与服务器间的代理服务器 异步）

  • 输入程序防抖
